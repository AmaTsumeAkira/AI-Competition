# CLI部署 Level 2：定制化·需求澄清

## 任务说明

你收到了一份模糊的需求文档，要求使用 Docker Compose 部署一个全栈应用。需求文档中包含多处矛盾和遗漏。

**你的任务：** 
1. 识别需求文档中的矛盾点和遗漏项
2. 撰写 `clarification_report.md` 澄清报告
3. 完成 Docker Compose 部署

## 需求文档（故意含歧义）

> 请部署一个电商系统后端，要求：
> 1. 支持高并发访问
> 2. 数据库需要加密存储
> 3. 部署到 K8s 环境（提供了 Docker Compose 文件模板）
> 4. 系统需要稳定可靠

## 交付物

1. `clarification_report.md` - 澄清报告（识别矛盾 + 建议方案）
2. `docker-compose.yml` - 部署配置
3. `deploy.sh` - 部署脚本
4. `healthcheck.sh` - 健康检查脚本

## 矛盾点（4 处）

1. "高并发"未定义具体 QPS
2. "数据库加密"未指定层级（传输层/存储层/字段级）
3. "部署到 K8s"但给了 Compose 目录
4. 遗漏日志收集方案

## 评分标准

- **澄清报告**：10 分（识别矛盾点质量）
- **部署完整性**：15 分（`docker compose up` 全部服务健康）
- **鲁棒性**：10 分（15 个测试用例）
- **满分**：35 分
- **限时**：60 分钟
