# CLI部署 Level 3：自动化工作流·抗破坏

## 任务说明

你收到了一个 Go 项目（~1200 行），需要为其构建完整的 CI/CD 自动化工作流。工作流必须能经受故障注入测试。

**你的任务：** 
1. 创建 Makefile（4 个构建目标）
2. 配置 CI 自动构建流水线
3. 实现蓝绿部署脚本
4. 实现回滚脚本
5. 通过 3 个故障注入测试

## 交付物

1. `Makefile` - 4 个构建目标（build/test/lint/deploy）
2. `ci.yml` - CI 配置文件
3. `deploy.sh` - 蓝绿部署脚本
4. `rollback.sh` - 回滚脚本

## 故障注入

| # | 故障类型 | 恢复要求 |
|---|---------|---------|
| 1 | 清空 `go.mod` 依赖版本 | 自动恢复依赖 |
| 2 | 部署中途 kill 进程 | 自动回滚到上一版本 |
| 3 | 注入编译通过但运行时 panic 的 commit | 检测并回滚 |

## 评分标准

- **基础功能**：20 分（Makefile 4 目标正常 + CI 构建通过）
- **CI 流水线**：10 分（自动触发 + 状态报告）
- **破坏恢复**：15 分（3 个故障注入，每个 5 分）
- **满分**：45 分
- **限时**：90 分钟
