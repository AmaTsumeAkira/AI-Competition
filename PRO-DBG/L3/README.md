# AI纠错 Level 3：系统级·连锁危机

## 任务说明

你收到了一个异步任务调度系统（订单处理流水线），基于 Python + Redis 模拟。项目包含 5 个模块、约 1800 行代码。系统包含 8 个 Bug，其中 3 组是连锁 Bug（修好一个会暴露另一个）。

**你的任务：** 使用 AI 工具定位并修复所有 Bug，通过安全扫描，提交完整修复报告。

## 文件清单

| 文件 | 说明 |
|------|------|
| `scheduler.py` | 任务调度器（~400 行） |
| `worker.py` | 工作线程（~300 行） |
| `api.py` | REST API 接口（~350 行） |
| `auth.py` | 认证模块（~200 行） |
| `config.py` | 配置管理（~150 行） |
| `test_runner.py` | 自动化测试（选手勿改） |

## Bug 清单

| # | 类型 | 描述 | 连锁 |
|---|------|------|------|
| 1 | 安全 | SQL 注入（搜索接口拼接用户输入） | - |
| 2 | 安全 | JWT 密钥硬编码 + 无过期时间 | - |
| 3 | 并发 | 库存超卖（并发竞态） | → 修后暴露 #4 |
| 4 | 连锁 | 修 #3 加锁后死锁（两接口不同序取两把锁） | - |
| 5 | 内存 | 日志中间件缓存请求体未轮转 | - |
| 6 | 配置 | staging/production 数据库串写反 | - |
| 7 | 安全 | 500 泄露堆栈含数据库凭据 | - |
| 8 | 业务 | 支付回调无幂等（重复入账） | - |
